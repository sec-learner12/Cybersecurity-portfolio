# OS Hardening & Brute Force Compromise Incident

This case study examines a security breach where a former employee gained unauthorized 
access to a website’s admin panel through a brute force attack on a weak default password. 
The attacker modified the site’s source code to redirect users to a malicious domain hosting 
malware. The incident highlights the importance of strong authentication controls and 
proactive OS hardening.

## Key Skills Demonstrated
- Web traffic and HTTP request analysis
- Identifying brute force attack indicators
- Understanding website defacement and malicious redirects
- Recommending OS hardening and authentication improvements

## Tools / Concepts Used
- HTTP GET request analysis
- Malware redirection behavior
- Password policy design
- MFA, CAPTCHA, and brute force mitigation strategies
